# PlaySport — Спортивный трекер (Учебный проект)

## Дисциплина
**Технологии и методы программирования**  
Практическая работа №6, МИИГАиК

## О проекте
**PlaySport** — это веб-приложение для отслеживания спортивной активности, построенное на архитектуре **MVC** с применением паттернов проектирования. Разработано в рамках учебной практической работы №6.

## Цели проекта
- Применение архитектурного паттерна **MVC**
- Использование паттернов проектирования:
  - **Структурные**: Adapter, Facade
  - **Поведенческие**: Observer, Strategy
  - **Производящие**: Factory, Singleton
  - **Параллельные**: Producer-Consumer (TaskQueue)
- Разработка модульной, масштабируемой и поддерживаемой системы

## Используемые технологии
- **C++** (серверная часть)
- **Vue.js 3** (клиентская часть)
- **CMake** (сборка)
- **HTML/CSS/JavaScript** (интерфейс)
- **Architecture**: MVC, Microservices-ready

## Паттерны проектирования
| Паттерн | Категория | Применение в проекте |
|---------|-----------|----------------------|
| **Singleton** | Производящий | Конфигурация (`Config`) |
| **Factory** | Производящий | Создание моделей (`ModelFactory`) |
| **Adapter** | Структурный | Адаптер данных (`DataAdapter`) |
| **Strategy** | Поведенческий | Стратегия аутентификации (`AuthStrategy`) |
| **Observer** | Поведенческий | Уведомления (`NotificationService`) |
| **Producer-Consumer** | Параллельный | Очередь задач (`TaskQueue`) |

## Сборка и запуск
### Требования:
- C++ компилятор (GCC/Clang/MSVC)
- CMake ≥ 3.10

### Сборка:
```bash
mkdir build
cd build
cmake ..
cmake --build .
```
### Запуск:
```bash
./playsport.exe
```

## Доступ к приложению
После запуска сервера приложение будет доступно по адресу:  
**http://localhost:8080**

## Функциональность

### Фронтенд (Vue.js 3)
- **Авторизация / регистрация**  
  Система аутентификации пользователей с валидацией данных
- **Дашборд с метриками**  
  Визуализация ключевых показателей активности пользователя
- **Управление тренировками**  
  Создание, редактирование, удаление и просмотр тренировок
- **Отслеживание целей**  
  Постановка и мониторинг спортивных целей с прогресс-баром
- **Лента активностей**  
  Хронологический просмотр всех спортивных активностей

### Бэкенд (C++ с архитектурой MVC)
- **Обработка HTTP-запросов**  
  REST API для взаимодействия с клиентской частью
- **Управление пользователями**  
  CRUD операции для пользовательских аккаунтов
- **Работа с тренировками и целями**  
  Бизнес-логика обработки спортивных данных
- **Аутентификация**  
  Система авторизации с токенами доступа
- **Очередь задач (TaskQueue)**  
  Асинхронная обработка фоновых операций
